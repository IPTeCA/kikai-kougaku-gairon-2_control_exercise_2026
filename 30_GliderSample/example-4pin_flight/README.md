# example-4pin_flight（元: 60_example-4pin_flight）

元ファイル：`SampleProgram/60_example-4pin_flight/60_example-4pin_flight.ino`

## これは何？

4サーボ（エレベータ、左右エルロン、ラダー）を想定した、より「機体制御らしい」スケッチです。

- 姿勢推定（Madgwick）で roll/pitch/yaw を得る
- 目標姿勢 `Q0` と比較して誤差 `E` を作る
- 各軸に対して PID を計算し、各サーボへ配分（ミキシング）して出力する

## 挙動（概略）

- `MANUAL`：Serialキーで各舵角を直接操作
- `AUTO`：目標姿勢に戻すようにPIDで舵角を生成
- `AUTO_TEST`：ゲインや目標軸を切り替えて挙動を見ながら調整するモード

※ 具体的なキー割り当ては、スケッチ冒頭のコメント（操作方法）にまとまっています。

## ここで入っている「工夫」

- 4舵のため、サーボごとに `servo_min/max` を持ち、出力をクリップしている（現実の可動範囲）
- ロール/ピッチ/ヨーでゲインや扱いが違う（特にヨーは実機の効きが弱かったり、計測が不安定になりやすい）
- 実機の段階に合わせてモード遷移し、パラメータの初期化やテストをしやすくしている

## 読み方（おすすめ）

1. `reset_parameters()`：どの変数が「状態」か（`E`, `SEdt`, `dE` など）を確認
2. 誤差計算：`E` / `SEdt` / `dE` の更新式を確認
3. サーボ出力：PIDの結果がどのサーボにどう入っているか（ミキシング）を確認
