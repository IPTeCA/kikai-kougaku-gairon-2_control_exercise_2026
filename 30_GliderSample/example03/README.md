# example03（元: 60_example03）

元ファイル：`SampleProgram/60_example03/60_example03.ino`

## これは何？

IMU（Madgwick）で roll/pitch/yaw（`Q`）を推定し、目標姿勢（`Q0`）に近づくようにサーボ角（`A`）を計算して動かす例です。

構造としては `E=Q0-Q` を使った **離散PIDの枠組み**になっています（ただし初期設定・キー操作では主に `Kp` を触るので、実質P中心で動かす想定です）。

## 挙動（モード）

- `MANUAL`：Serialキーで `A0`（サーボの基準角）を動かす
- `AUTO`：姿勢誤差 `E` から `A = A0 + Kp*E + Ki*∫E dt + Kd*dE/dt` を計算してサーボに反映
- `START` → `REAL_FLIGHT`：パラメータ初期化の後に自動制御へ移行（同じ制御計算を継続）

## どうやってモードを選ぶ？

本体の向きを使った「モード選択」が入っています（`mode_select()` → `detect_mode()`）。

- 加速度 `Acc(2)` が一定条件のときに数秒保持すると、加速度の向き（どの軸が一番大きいか）でモードが決まる仕組みです。

## キー操作の意味

- `MANUAL` 中は、各軸の `A0` を増減（= サーボの基準角を動かす）
- `AUTO` 中は、主に `Kp` を増減（= 姿勢を戻す強さを調整する）

## 学びどころ

- 角度推定（`Q`）→ 誤差（`E`）→ 操作量（`A`）→ サーボ、という流れがはっきり読める
- `dt` を使って I/D を作っている（`errors()`）
- サーボの飽和（±90→0..180）を考慮している
