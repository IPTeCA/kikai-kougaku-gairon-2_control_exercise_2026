# example04（元: 60_example04）

元ファイル：`SampleProgram/60_example04/60_example04.ino`

## これは何？

`example03` を発展させ、実機で問題になりやすい点（センサオフセット、積分の扱い）に対して改善を入れた例です。

## 追加された工夫（何が変わった？）

### 1) センサのオフセット補正（キャリブレーション）

`calibration()` で一定回数サンプリングして平均を取り、

- 加速度：`Acc0`
- 角速度：`Gyr0`

を求め、読み取り値から引いています。  
静止していてもゼロにならないバイアス（オフセット）を抑える狙いです。

### 2) 積分の作り方を2系統にしている

- `SEdt0`：台形則の「普通の積分」に近い更新
- `SEdt`：過去の誤差バッファ `err[][]` を使った更新（時間窓を意識した“積分成分”）

制御式の中で

- `KI*SEdt + KI0*SEdt0`

のように複数のI成分を混ぜています。

## なぜこうする？

- 実機ではオフセットやドリフトで「ずっと少しズレている」状態が起きやすい
- I項は有効ですが、入れ方次第で暴れやすいので、挙動を見ながら成分を分けて調整できるようにしている
