# example05（元: 60_example05）

元ファイル：`SampleProgram/example05/example05.ino`

## これは何？

`example04` をさらに発展させ、特に **D項（微分）や角度のノイズ**に対して安定化する工夫を入れた例です。

## 追加された工夫（何が変わった？）

### 1) 角度 `Q` をそのまま使わず、一次遅れフィルタ後の `QQ` を使う

`errors()` 内で

- `QQ = (フィルタ係数) * Q + ...`

のようにして、角度を平滑化した値 `QQ` を作り、

- `E = Q0 - QQ`

として誤差を計算しています。

### 2) 微分（`dE`）をフィルタ付きの形で更新

単純な `(E - E_prev)/dt` ではなく、時定数 `Tc` を使った更新式にして、ノイズで `dE` が暴れにくいようにしています。

## なぜこうする？

- 角度推定（姿勢推定）や誤差にはノイズが乗り、D項はそれを増幅しやすい
- 「角度（計測値）」や「微分項」に軽いローパスを入れると、サーボがビリビリ動く症状が減ることがあります

## 注意

フィルタを強くし過ぎると位相遅れが増えて、応答が鈍くなったり不安定化することもあります。`Tc` の意味（カットオフ周波数）を意識して調整します。
