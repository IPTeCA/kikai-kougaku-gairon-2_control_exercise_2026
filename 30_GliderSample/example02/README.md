# example02（元: 60_example02）

元ファイル：`SampleProgram/60_example02/60_example02.ino`

## これは何？

`example01` を発展させて、**教科書通りの離散PID**の形（P/I/Dの項を持つ形）をスケッチ上で揃えた例です。

このプログラムでは、誤差を

- `E = Q0 - Q`

として、次を逐次更新します。

- 積分（台形則）：`SEdt += (E + E_prev)/2 * dt`
- 微分（差分）：`dE ≈ (E - E_prev)/dt`

そして操作量（サーボ角の差分）を

- `A = A0 + mode * (Kp*E + Ki*SEdt + Kd*dE)`

として計算します。

## 挙動

- `mode=0`：`A = A0` のみ（手動の角度オフセット操作）
- `mode=1`：姿勢角が目標からずれたら戻す（PID枠組み）

## なぜこの段階が必要？

`example03` 以降のプログラムは、最終的に「誤差→P/I/D→サーボ」というパイプラインに乗っています。
この段階で、変数（`E`, `SEdt`, `dE`, `dt`）の意味を揃えておくと読みやすくなります。

## 実装上の注意

- `dt` は**秒**で計算する前提です（`millis()` 等から換算）。`dt` が0に近い/0になると微分項が壊れます。
- サーボ角は飽和するので、本来は **アンチワインドアップ**（Iの上限・飽和時の積分停止など）が必要になります。
